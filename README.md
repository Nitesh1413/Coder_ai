# 🚀 Coder AI: An Autonomous, Self-Healing Code Generation & Execution Engine

## 🌐 Overview
**Coder AI** is a high-performance, localized AI agentic system designed to bridge the gap between "Generative AI" and "Functional Software." While most AI models simply provide text-based code snippets that may or may not work, **Coder AI** takes a radical approach: it writes the code, executes it in a restricted environment, captures errors, and self-corrects until the code is verified as functional.

In its current state, the project serves as a **robust architectural blueprint**. It is configured in **Test Mode**, using high-fidelity dummy responses to demonstrate the complex "Self-Healing Loop" and "Docker Sandboxing" without requiring immediate API expenditures.

---

## 🎯 What Does This Project Do?
At its core, Coder AI is a **Private Software Engineer** residing on your local machine. Its primary functions include:

1.  **Natural Language Processing:** Translating complex user requirements into executable Python logic.
2.  **Isolated Execution:** Running untrusted, AI-generated code inside a **Docker Container** (Alpine Linux) to ensure the host system remains 100% safe.
3.  **Real-Time Debugging:** Capturing `STDOUT` (output) and `STDERR` (errors) from the execution environment.
4.  **The Self-Healing Loop:** If a script fails, the agent analyzes the traceback, understands the root cause of the crash, and rewrites the script autonomously.
5.  **Resource Management:** Enforcing strict hardware limits (e.g., 30MB RAM) to ensure the agent remains lightweight and non-disruptive to the host machine.

## 🏗️ The Development Journey: How I Built This
Building Coder AI wasn't just about writing scripts; it was about designing a **Reliability Framework**.

### 1. The Architectural Vision
I noticed that LLMs (Large Language Models) often suffer from "Hallucinations"—writing code that looks correct but fails due to syntax errors or missing dependencies. I decided to build a "Supervisor" for the AI. I chose **FastAPI** for the backend because of its asynchronous capabilities, allowing the system to handle long-running Docker processes without freezing the UI.

### 2. Solving the Security Paradox
Running code generated by an AI is inherently risky. To solve this, I integrated **Docker**. By creating a "Sandbox," I ensured that even if the AI generates a malicious script (like `os.remove('/')`), it only affects a temporary, 5MB virtual environment that is deleted seconds later.

### 3. Implementing the Feedback Loop
The hardest part was the "Self-Healing" logic. I had to design a recursive loop that could:
* Identify if a failure was due to logic or environment.
* Feed the *exact* error message back to the AI.
* Limit retries to prevent infinite loops (cost management).

## 💎 Code Specialties & Technical Breakdown

### 1. `main.py` (The Orchestrator)
This is the heart of the system. It handles the **API Lifecycle**.
* **Asynchronous Flow:** Uses `async/await` to ensure that while Docker is spinning up, the server remains responsive.
* **State Management:** It tracks the number of "Attempts" and manages the transition between the AI and the Sandbox.

### 2. `ai_agent.py` (The Brain)
This module interfaces with **Google Gemini 2.0 Flash**.
* **System Prompting:** I designed a strict system instruction that forces the AI to output *only* raw Python code, stripping away markdown backticks (```python) which usually break execution.
* **History Awareness:** It doesn't just send the prompt; it sends the *Error History*, allowing the AI to learn from its previous mistake in the same session.

### 3. `sandbox.py` (The Jailer)
This is where the magic of security happens.
* **Ephemeral Containers:** It creates a container, runs the code, and deletes the container in milliseconds (`auto_remove=True`).
* **Memory Capping:** Strictly limited to **30MB RAM** using the `--memory` flag in Docker.
* **Temporary Filesystem (`tmpfs`):** All files created by the AI are stored in RAM, not on your Hard Drive, protecting your SSD's lifespan.

## 📥 Installation & Local Setup Guide

If you wish to install Coder AI on your system, follow these precise steps:

### Phase 1: System Requirements
* **Operating System:** Windows 10/11 (PowerShell recommended), macOS, or Linux.
* **Docker Desktop:** This is mandatory. The project will not run without a functional Docker Daemon.
* **Python 3.10+:** Ensure Python is added to your System Path.

### Phase 2: Deployment
1.  **Clone the Repository:**
    ```powershell
    git clone https://github.com/Nitesh1413/Coder_ai.git
    cd Coder_ai
    ```
2.  **Environment Setup:**
    ```powershell
    python -m venv venv
    .\venv\Scripts\activate
    ```
3.  **Installing Dependencies:**
    ```powershell
    pip install -r requirements.txt
    ```
4.  **Docker Preparation:**
    Open Docker Desktop and run:
    ```powershell
    docker pull python:3.10-alpine
    ```

## ❓ Developer's FAQ (Internal Interaction Log)
During the development of Coder AI, several critical questions were addressed to ensure the project's viability:

**Q: Will the 30MB Docker memory limit affect my laptop's performance?**
**A:** No. Your laptop (8GB RAM) has ~8000MB. 30MB is less than 0.4% of your total RAM—equivalent to a single empty Chrome tab. It is highly efficient.

**Q: Where does the generated code go after execution?**
**A:** By default, the code is returned as a JSON response. However, the system is designed to "Auto-Save" successful scripts into a `generated_code.py` file in the root directory for permanent storage.

**Q: Why use Python-Alpine in Docker?**
**A:** Size. A standard Python image is ~900MB. Alpine is ~50MB. This ensures the agent starts up and executes code in under 2 seconds.

**Q: Can this AI delete my files?**
**A:** No. Because the code runs inside a Docker Sandbox, it has no access to your `C:` drive or `D:` drive unless you explicitly mount them. It is a "Zero-Trust" architecture.

## 🛠️ Detailed Troubleshooting Guide

| Symptom | Probable Cause | Resolution |
| :--- | :--- | :--- |
| `FileNotFoundError: main` | Terminal is in the wrong directory. | Run `cd OneDrive\Desktop\coder_ai` to enter the project folder. |
| `Scripts disabled on system` | Windows PowerShell security policy. | Run `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser`. |
| `No module named 'uvicorn'` | Library installed globally, not in venv. | Activate venv first, then run `pip install uvicorn`. |
| `Docker API connection failed` | Docker Desktop is closed. | Launch Docker Desktop and ensure the whale icon is green. |
